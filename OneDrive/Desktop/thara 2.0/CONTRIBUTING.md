# Contributing to Purple 2.0

Thank you for your interest in contributing to Purple 2.0! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork:
```bash
git clone https://github.com/YOUR_USERNAME/purple-2.0.git
cd purple-2.0
```
3. Create a new branch for your feature:
```bash
git checkout -b feature/your-feature-name
```

## Development Setup

1. Install dependencies:
```bash
pip install -r requirements.txt
```

2. Ensure all model files are present or generate them:
```bash
python model_train.py
```

## Code Style

- Follow PEP 8 Python style guide
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions focused and single-purpose

## Commit Guidelines

- Write clear, descriptive commit messages
- Reference issue numbers if applicable
- Use present tense ("Add feature" not "Added feature")

## Pull Request Process

1. Update the README.md if needed
2. Add tests if applicable
3. Ensure all code works on Windows 10/11
4. Submit a pull request with a clear description

## Reporting Issues

When reporting bugs, please include:
- Python version
- Operating system version
- Error messages/logs
- Steps to reproduce

## Feature Requests

For new features, please describe:
- The feature you'd like
- Why it would be useful
- How you envision it working
